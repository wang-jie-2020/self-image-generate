## 首页：这个 Skill 怎么工作？ 🧭

- 它不是直接开画，而是先判断输入模式
- 它用 5 个参数控制图解数量、风格和表达深度
- 它会先把输入整理成概念、关系和叙事顺序
- 它有两道人类确认点，避免画偏
- 最后才生成 Mermaid，并按清单验证

---

## 图 1: 总体工作流 🚦

```mermaid
flowchart TD
    A(["用户触发 skill"]) --> B{"选哪种模式？"}
    B -->|"调研话题"| C["调研模式"]
    B -->|"已有文本"| D["文本模式"]
    C --> E["收集 5 个参数"]
    D --> E
    E --> F["处理输入内容"]
    F --> G["提取概念和关系"]
    G --> H["设计图解方案"]
    H --> I{"用户确认方案？"}
    I -->|"要调整"| H
    I -->|"可以"| J["生成 ASCII 骨架"]
    J --> K{"用户确认骨架？"}
    K -->|"要调整"| J
    K -->|"可以"| L["生成 Mermaid 图解"]
    L --> M["验证并输出文件"]
```

这张图是总览：真正的 Mermaid 生成其实在很后面，前面大部分时间都在理解、设计和确认。

## 图 3: 参数控制台 🕹️

```mermaid
mindmap
  root((生成参数))
    生成数量
      控制图表张数
      控制覆盖深度
    炫酷程度
      控制 emoji 密度
      控制视觉装饰
    简洁程度
      控制节点字数
      控制说明长度
    配色方案
      L3 以上生效
      L1 L2 不加颜色
    目标受众
      控制术语难度
      控制解释深度
```

这 5 个参数像一个控制台：数量决定画多少，炫酷决定视觉密度，简洁决定节点文字，受众决定说话方式。

## 图 4: 双重人工确认 🧑‍⚖️

```mermaid
flowchart TD
    A["结构化结果"] --> B["生成图解方案"]
    B --> C{"确认点 1：方案对吗？"}
    C -->|"不对"| D["调整画什么和怎么画"]
    D --> B
    C -->|"对"| E["生成 ASCII 骨架"]
    E --> F{"确认点 2：骨架对吗？"}
    F -->|"不对"| G["调整节点和连接"]
    G --> E
    F -->|"对"| H["正式生成 Mermaid"]
```

这就是这个 skill 很关键的地方：先确认“画什么”，再确认“大概长什么样”，最后才动手生成正式代码。

## 图 5: 用户、Claude、参考文件怎么协作 💬

```mermaid
sequenceDiagram
    participant U as 用户
    participant C as Claude
    participant S as Skill 指令
    participant R as Reference 文件
    participant O as 输出文件

    U->>C: 触发 skill 并给出任务
    C->>S: 读取工作流程规则
    C-->>U: 收集生成参数
    U-->>C: 确认参数
    C->>R: 读取文本或调研指南
    C-->>U: 汇报分析结果
    U-->>C: 确认继续
    C->>R: 读取图表模式和样式规则
    C-->>U: 给出方案和 ASCII 预览
    U-->>C: 确认生成
    C->>O: 写入 Mermaid Markdown 文件
    C-->>U: 返回路径和参数回顾
```

这张图强调交互顺序：用户不是只在开头说一句话，而是在关键节点参与确认。

## 图 6: 生成与验证闭环 ✅

```mermaid
flowchart TD
    A["按方案生成 Mermaid"] --> B["检查中文节点引号"]
    B --> C["检查 emoji 和文本长度"]
    C --> D["检查图表类型限制"]
    D --> E["检查参数是否一致"]
    E --> F{"全部通过？"}
    F -->|"没有"| G["修正问题"]
    G --> A
    F -->|"通过"| H["写入 diagrams 文件"]
    H --> I["输出参数回顾"]
```

最后一步不是写完就算了，还要按清单检查：语法、参数、风格、数量和叙事连贯性都要对上。
